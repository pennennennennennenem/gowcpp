## 口上

気付けば30年もの間、学業やゲームや仕事の合間にフリーソフトウェアを作っては公開する日々を過ごしておりました。

GOwはその中でも最初に公開されたソフトウェアであり、なんと前世紀から、ぼくの毎日のマイコン生活に欠かせないソフトウェアでしたし、これからもそうでしょう。

最近はAIの力を借りてプログラミングをする事が出来るようになり、Win32 APIを使ってちょっとした小物を作る事も従来ほどの苦行ではなくなりましたから、丁度よい機会であるとしてWin32 APIとSTLで書き直したバージョンを作ってみました。


## 概要

explorerからのローカルのディレクトリへのアクセスを素早く行うためのユーティリティです。

例えば `d:\game\touhou\kaeizuka` にアクセスしたいと思ったら、ショートカットを使わずマイコンピュータから辿る場合、アイコンを探してダブルクリックして、という作業を5回行わなければなりません。

一方、GOwを用いた場合、デフォルトでは`Ctrl+Alt+G`に設定してあるホットキーで検索ボックスを表示したら`kaei`とでも入力してEnterキーを押せば、もうデスクトップにお目当てのフォルダが表示されるでしょう。

このように、最後のディレクトリ名部分の先頭部分を入力すればキャッシュされたフルパス一覧から名前方一致検索してexplorerで開いてくれる、という寸法です。


## インストール方法・アンインストール方法

releaseに置いてあるgowcpp.zipを解凍して任意のフォルダを掘ってコピーすれば、もうインストールは完了です。

気に入ったらスタートアップフォルダ(`win+R`で`shell:startup`と打つと出てきます)にgowcpp.exeへのショートカットを追加しとくと良いでしょう。

アンインストールするにはインストールしたフォルダごと削除すればシステムから跡形もなく消えます。スタートアップフォルダにショートカットを入れてたらそれも消しとけばいいですね。

レジストリへの書き込みは行いませんし、勝手にサービスをインストールするなど邪悪な振る舞いもいたしません。


## 使用方法

gowcpp.exeを起動するとタスクトレイにアイコンが追加されます。

トレイアイコンを右クリックして、`Quit`を選択するとプログラムを終了するほか、トレイアイコンにマウスオーバーする事で、後述のディレクトリスキャンの進行状況を確認できます。

`Ctrl+Alt+G`を押すと、検索ボックスが表示されます。普段よく使うドライブがC:トライブ以外、例えばD:ドライブの場合は、`D:`と入力してEnterキーを押すと、使用するドライブをD:ドライブに変更できます。

Enterキーを押すと検索ボックスは一旦閉じるので、もう一度`Ctrl+Alt+G`で検索ボックスを開くことが出来ます。

次に、ドライブの中で飛びたいディレクトリがある場合は、その名前を打つと検索…してくれますが、インストールしたばかりの状態ではドライブの情報がまだ無いので、先にディレクトリスキャンをするか聞かれます。`はい`を選択してしばらく待ちましょう。ディレクトリが多いほど時間が掛かり、数分を要することもあります。この操作によって、使いたいドライブにどんな名前のディレクトリがあるのか、という情報を網羅します。

右下にスキャンが完了した旨バルーンが表示されたら、explorerで開かせたいディレクトリ名を打ち込んで検索させてください。

`Ctrl+Alt+G`以外に、`Ctrl+Alt+C`を押して検索ボックスを開いた場合、検索結果のパスをクリップボードにコピーする機能も有ります。

### 同一の名前を持ったディレクトリが多すぎる場合

`texture`などの良くあるフォルダ名に飛びたいけどリストに検索結果のリストに山のように出てきてしまう…という場合、先祖のディレクトリ名を入れることで絞り込み検索を行うことが出来ます。

例えば、`work\texture`と打つと、先祖にあたるディレクトリのどこかで`work`から始まるディレクトリを持ち、かつ末端のディレクトリ名が`texture`から始まるディレクトリが検索されます。`work\a\texture`のように多段階の検索も可能です。

### フルパスを直接打った方が早い場合

`\work`など、ルートディレクトリに近いフォルダを開く場合には直接フルパスで打った方が早いでしょう。`\`から検索ボックスに書いた場合、フルパス名が直接指定されたとしてexplorerで開きます。検索は行いませんが、入力中に`TAB`キーを押すと、候補が一つに絞られる場合は補完されます。


### ディレクトリスキャンについて

ディレクトリの追加・削除などが行われた場合は、`Ctrl+Alt+S`を押すことで手動でスキャンを行う必要があります。これにより、gowcpp.exeのあるディレクトリに`cache`ディレクトリがあり、その中に各ドライブについてのフルパス一覧が作成されます。

ディレクトリスキャンはドライブごとに行う必要があるので、始めて検索の対象にするドライブを使う場合は、その都度手動でスキャンするか、先ほどのダイアログに応答することでスキャンを開始させましょう。

ディレクトリスキャン中でも使用するドライブについてのキャッシュが既にある場合には、普通に検索してexplorerで開かせる機能などは使うことが出来ます。

### ラウンチャとしてつかう

ホットキーにコマンドを割り当ててアプリケーションを起動する事も出来ます。デフォルトの設定では以下のように割り当てられています。

- `Ctrl+Alt+H` 入力文字列でGoogle検索
- `Ctrl+Alt+L` 電卓を起動します
- `Ctrl+Alt+T` タイマーを起動します
- `Ctrl+Alt+W` 入力文字列でWikipediaを検索

## gowcpp.iniによる設定方法

gowcpp.iniを適宜書き換えてコマンドや起動用のホットキーを追加・削除・変更することが出来ます。一度書き換えると滅多にいじらないので設定用のUIなどはありません。メモ帳で十分。

以下のように、`item`から始まる名前を持ったセクションがホットキーによって起動されるコマンドであると認識されます。


```
[item0]
desc = find directory
hotkey = Ctrl+Alt+G
file = explorer
param = %s
```

`hotkey`にホットキーの組み合わせと`file`に実行するプログラムを記述します。`param`にはプログラムに渡されるパラメータを指定します。

`file`または`param`には`%`から始まるマクロを指定することが出来、以下のマクロをサポートしています。

|マクロ|動作|
|--|--|
|`%s`|検索ボックスで検索した結果のフルパス名に置換されます|
|`%c`|検索ボックスで検索した結果のフルパス名に置換され、クリップボードにコピーします|
|`%d`|検索ボックスの入力内容に置換されます|
|`%u`|検索ボックスの入力内容をURLエンコードした文字列に置換されます|
|`%%`|%に置換されます|


また、`file`には以下の予約語に対応した動作があります。

|予約語|動作|
|--|--|
|`*SCAN`|使用中のドライブをスキャンする(`param`は無視されます)|

日本語など半角英数字以外の文字列を使う場合は、iniファイルをUTF-16で保存してください。UTF-8ではないのでご注意を。

`desc`は検索ボックスに表示されるタイトルとして使われます

### 検索除外ディレクトリの設定

ゴミ箱の中を検索結果に含めたくない場合などに便利なように、除外リストを作れます。以下のように、`[exclude]`セクションに`dir`から始まるエントリを追加すると、そのディレクトリ以下は検索結果より除外されます。

```
[exclude]
dir0 = D:\foo\bar\hogehoge
```

この場合、Dドライブを使用している場合に、`\foo\bar\hogehoge\`というディレクトリに完全に一致するディレクトリとそのサブディレクトリは検索結果に現れません。リムーバブルメディアに割り当てられるドライブについて除外リストを使うことはあまり想定してません。

```
[exclude]
dir1 = \$recycle.bin
```

ドライブレターを省略すると、どのドライブを使っている場合でも除外対象として扱われます。

### LRUリストの数の設定

直近の検索結果をドライブごとに保存するLRUリストを保持でき、リスト内にあるパスは検索結果の上に置かれます。

エントリの数は`[config]`セクションの`LRU`で設定出来ます。LRUリスト自体必要ない場合は0を設定しておけばいいでしょう。


### 検索結果表示リストが狭い/広い場合

`[config]`セクションの`ListWidth`で検索結果表示リストの幅を検索ボックスに対する相対値で指定できます。単位はピクセルです。検索ボックスの大きさは固定です。
